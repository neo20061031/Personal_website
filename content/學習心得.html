W3:什麼是 git, GitHub, gist 與 cmsimde? 如何使用這些工具？為什麼要使用這些工具？ (動態模擬)

A:Git 是一套幫你紀錄檔案修改歷史的工具，主要處理程式碼或筆記這類純文字檔，讓你能對比哪裡改過，或是退回舊版本。GitHub 則是放這些專案的雲端平台，除了備份和找人一起寫，還能拿來架個人網站，只是檔案大小有限制。 Gist 也是 GitHub 的功能，讓你快速分享幾行程式碼或小筆記，背後一樣是用 Git 在跑，但它就沒辦法傳圖片這類的二進位檔案。CMSIMDE 是把上面這些東西通通整合起來的管理介面，讓你可以直接用 Edit 改內容、用 Convert 轉成網頁，最後按 acp 按鈕，系統就會自動跑完 git add、commit 跟 push，把檔案推上 GitHub 發佈網站。用這些工具的好處就是修改紀錄很完整、備份也安全，而且不用自己手打那些複雜的指令。(這對於不會打代碼的我有很大的幫助，就如同此時此刻我能打好字按Save>Convert>acp就能直接完成網站編輯和上傳，很方便)

W7:Brython 是什麼？如何在瀏覽器配置 Brython 環境？ Brython 能做什麼？ (Brython, Examples)

A:Brython 是能讓你在網頁上直接寫 Python 的工具。一般的網頁通常都要用 JavaScript 來寫，但 Brython 會把你的 Python 程式碼轉成瀏覽器看得懂的語法。設定環境需要三步，在 HTML 網頁頭部引入 brython.js 這個檔案，然後在頁面載入時啟動 brython() 函式，最後把的 Python 程式碼包在專用的 <script type="text/python"> 標籤裡就行了。Brython 幾乎能做所有網頁上的事，像是改字、改顏色、處理按鈕動作，甚至還能直接用 Python 的數學庫來算東西。目錄裡的機器人模擬（brython_robot.py）就是很好的例子，它就是用 Python 語法在網頁上控制機器人跑來跑去。
